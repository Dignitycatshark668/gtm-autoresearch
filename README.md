## 🚀 What this app does

gtm-autoresearch helps you run a repeatable research loop for SaaS scoring models. It keeps model weights under version control, stores each experiment in an append-only journal, and tracks metrics that fit go-to-market work.

Use it to:
- test churn and health score ideas
- compare scoring changes over time
- keep a clean record of each run
- review results without sorting through loose files

## 🔎 How to use it

1. Start a new research run.
2. Load the customer or account data you want to score.
3. Pick the model or rule set you want to test.
4. Run the scoring loop.
5. Review the output metrics.
6. Save the run so you can compare it later.

Each run should help you answer questions like:
- Which accounts look at risk?
- Which signals matter most?
- Did the score improve after the change?
- How did the model affect NRR or churn views?

## 📊 What you can track

It also helps you keep:

## 🛠️ Common tasks

### Start a fresh experiment
- open the app
- create a new run
- choose your data
- set the score rules or model version
- run the analysis

### Compare two versions
- open the journal
- select two saved runs
- review the score changes
- check whether the newer version improves results

### Review past work
- open the experiment journal
- find the run you want
- read the saved notes and output
- use the record to repeat or adjust the test

## 🧩 Best way to use the app

For clean results:
- use one project folder per test area
- keep each run named with the date
- save a note about what changed
- do not overwrite old results
- keep weights and data in the same project path

A simple naming pattern works well:
- `2026-04-28-baseline`
- `2026-04-28-new-signals`
- `2026-04-28-weight-update`

## 🧪 Example workflow

1. Load your current customer list.
2. Run the baseline health score.
3. Save the output.
4. Update one signal or weight.
5. Run the same data again.
6. Compare the two runs.
7. Keep the better version and note the reason.

This makes it easier to see which changes help and which changes do not.

## 🖥️ Troubleshooting

### The app does not open
- make sure the download finished
- check that you extracted the files if they came in a ZIP
- try running the file again
- restart Windows and try once more

### Windows blocks the file
- right-click the file
- choose Run anyway if that option appears
- confirm you downloaded it from the GitHub page above

### The app cannot find my files
- check that you did not move part of the project folder
- keep data, weights, and journal files together
- use a short folder path like `C:\GTM-Scoring`

### The screen looks empty
- load a project or sample data set
- start a new run
- check that the app opened the right folder
